Class Detail
ttrace.classes.TraceNodeEx(parentNode, generateUniqueId)
Parameters:
{string} parentNode
The parent node where to place that trace. The IconIndex and the enabled properties are also recopied Parameters can be null : the root tree become the parent node, enabled is true and the default icon is used
{string} generateUniqueId
if true, the id is generated automatically, else set the empty string
Field Detail
enabled
When enabled is false, all traces are disabled. Default is true. All node have a enabled property, that lets you define group of enabled trace. For example set the TTrace.Debug.enabled to false but continue to accept Error and Warning traces
fontDetails
fonts details applied to the member. Don't use it directly. Use the addFontDetail function in place
iconIndex
The index of the icon to use. You can then show an icon for Warning traces different for Error traces
id
The unique ID. Normally it's a GUID, but can be replaced by something else for inter process traces.

Left message
